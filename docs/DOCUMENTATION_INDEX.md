## üìù Documentation Standards

### Creating New Documentation

When creating new documentation, follow these guidelines:

1. **Use consistent formatting**
   - Title with `#`
   - Metadata (Last Updated, Status, etc.)
   - Table of Contents for long docs
   - Clear section headers

2. **Include essential sections**
   - Overview/Purpose
   - Prerequisites (if applicable)
   - Step-by-step instructions
   - Examples
   - Troubleshooting
   - References

3. **Keep it updated**
   - Add "Last Updated" date
   - Update when making changes
   - Archive outdated docs

4. **Link related docs**
   - Cross-reference related documentation
   - Use relative links
   - Keep navigation easy

### Documentation Naming Convention

```
CATEGORY_NAME.md          - General category docs
MODULE_NAME_MODULE.md     - Module-specific docs
PROCESS_NAME_PROCESS.md   - Process documentation
GUIDE_NAME_GUIDE.md       - Tutorial-style guides
```
